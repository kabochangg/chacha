# AGENTS.md

## プロジェクト方針

このリポジトリは、Webアプリゲーム「勇者のあとしまつ / ダンジョン清掃員の生活」を開発するためのプロジェクトです。

旧「副業タイプ診断」サイトの仕様は引き継ぎません。今後の判断は、ゲーム仕様書とこのドキュメントを優先してください。

## 最優先する体験

実装では、次の順に優先します。

1. スマホで操作しやすいこと
2. 掃除アクションが気持ちいいこと
3. 素材回収と報酬獲得が分かりやすいこと
4. 1回3〜5分で終わること
5. 軽く動くこと

画面の豪華さ、複雑なシステム、オンライン機能、長いストーリーよりも、短いゲームループの完成度を優先してください。

## 技術方針

- 言語はTypeScriptを使う
- ゲームエンジンはPhaser.jsを使う
- ビルドはViteを使う
- スタイルはCSSで管理する
- セーブはlocalStorageを使う
- PWA対応を行う
- 初期公開先はCloudflare Pagesとする

MVPでは、サーバー、DB、ログイン、ランキング、課金、クラウドセーブ、マルチプレイ、チャット、リアルタイム通信を実装しません。

## 画面と入力

- スマホ縦画面を最優先する
- PCでもキーボード操作できるようにする
- スマホ操作は左下の固定仮想スティック、右下の2ボタンを基本にする
- 右下の大ボタンは掃除、小ボタンは回避にする
- 掃除は長押し操作にする
- 「調べる」は常時ボタンにせず、宝箱、出口、素材、特殊オブジェクトに近づいた時の状況アクションとして扱う
- 素材は近づいたら自動回収する
- メニューは右上の一時停止ボタンで開く
- MVPの操作は移動、長押し掃除、回避、状況アクション、メニューに絞る
- ボタン数を増やしすぎない
- タップ範囲は広めに取り、誤操作を減らす
- 主要ボタンのタップ領域は64px以上、その他のタップ領域は48px以上を目安にする
- タッチ操作中にブラウザスクロールが起きないようにする
- iPhoneのホームインジケータなど、端末のセーフエリアを考慮する
- 横持ちはMVP対象外とし、縦画面で崩れないことを優先する

掃除操作の基本:

- 残骸に近づいて掃除ボタンを長押しすると清掃する
- 対象はプレイヤーが向いている方向、または最寄りの残骸を優先する
- 長押し中は進捗リング、短い揺れ、粉じん、掃き音で手応えを出す
- 長押しを離すと清掃を中断する
- 耐久値が低い残骸はすぐ消し、耐久値が高い残骸は複数拍かけて消す

PC操作の基本:

- `W / A / S / D`: 移動
- `方向キー`: 移動
- `Space`: 掃除
- `Shift`: 回避
- `E`: 調べる
- `Esc`: メニュー

## 実装範囲

MVPでは次を実装対象にします。

- タイトル画面
- 拠点画面
- ダンジョン画面
- リザルト画面
- プレイヤー移動
- 掃除アクション
- アイテム回収
- HPまたはスタミナ
- 所持金
- 素材
- localStorageセーブ
- ほうき強化
- バッグ強化
- PWA
- Cloudflare Pages公開

MVPでは、ダンジョンは「はじまりの地下道」1つに絞ります。強化対象は、清掃力に関わる「ほうき」と、所持容量に関わる「バッグ」を優先します。

## データ方針

- アイテム、ダンジョン、強化データはTypeScriptのデータ定義として管理する
- セーブデータには `version` を含める
- セーブデータの読み込み失敗時は初期データに戻せるようにする
- 将来のDB化を見越して、保存形式は単純なJSONにする

## ファイル編集ルール

- 文字コードはUTF-8を使う
- 日本語Markdownは文字化けしないように確認する
- 既存のユーザー変更を不用意に戻さない
- コードや構成を変更するときは、目的に関係する範囲だけを編集する
- 旧診断サイト由来の広告、問い合わせ、AdSense、ASP運用はゲームMVPには持ち込まない
- `Skills.md` のファイル名は維持する

## 確認コマンド

実装後は、プロジェクト構成に応じて次を確認します。

```powershell
npm install
npm run dev
npm run build
```

テストが追加された場合は、該当するテストコマンドも実行します。

ドキュメントだけを変更した場合は、少なくとも次を確認します。

```powershell
git diff -- README.md PLAN.md AGENTS.md Skills.md
```

## 受け入れ基準

ゲームMVPは、次を満たしたら完成とします。

- タイトル画面からゲーム開始できる
- 拠点からダンジョンに入れる
- プレイヤーを操作できる
- 残骸を掃除できる
- 素材を拾える
- 出口から帰還できる
- リザルトが表示される
- 所持金が増える
- ほうきまたはバッグを強化できる
- ブラウザを閉じても続きから遊べる
- スマホで操作できる
- 移動しながら掃除ボタンを押し続けられる
- 仮想スティック操作中に画面スクロールが起きない
- Cloudflare Pagesで公開できる

最初の成功条件は、プレイヤーが「もう一回だけダンジョンに行きたい」と感じることです。
