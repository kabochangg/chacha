# AGENTS.md

## プロジェクト方針

このリポジトリは、Webアプリゲーム「勇者のあとしまつ / ダンジョン清掃員の生活」を開発するためのプロジェクトです。

旧「副業タイプ診断」サイトの仕様は引き継ぎません。今後の判断は、ゲーム仕様書、このドキュメント、`PLAN.md`、`DESIGN.md` を優先してください。

現在の目標は、MVPを磨き込み、v1公開版としてCloudflare Pagesで公開できる完成度にすることです。長期運用版ではなく、最初に人へ遊んでもらえる短時間清掃ゲームとして完成させます。

## 最優先する体験

実装では、次の順に優先します。

1. スマホで操作しやすいこと
2. 掃除アクションが気持ちいいこと
3. 素材回収と報酬獲得が分かりやすいこと
4. 1回1〜3分で終わること
5. 軽く動くこと

画面の豪華さ、複雑なシステム、オンライン機能、長いストーリーよりも、短いゲームループの完成度を優先してください。

## v1公開版の判断基準

- v1では「はじまりの地下道」をB1F〜B5F構成にし、B5Fを最奥地にする。
- 新ダンジョン追加より、操作、清掃、回収、報酬、強化の気持ちよさを優先する。
- 各出撃ごとにマップ配置とスタート位置をある程度ランダムにする。
- 各フロアのモンスター数は2〜5体の範囲でランダムにする。
- ダンジョンに行くごと、または深い階へ行くごとにモンスターを少し強くする。
- モンスターは視線扇形を持ち、プレイヤーに気づいたら `!` を出して追跡する。
- 添付イメージは完成形の方向性として扱うが、画像そのものを素材として直貼りしない。
- 高密度ピクセルアート風を目指す場合も、スマホで読めて軽く動くことを優先する。
- 図鑑、持ち物、依頼、ショップはv1相当の最低限機能を入れる。
- 空のタブや壊れた導線を残さない。
- 最初の成功条件は、プレイヤーが「もう一回だけダンジョンに行きたい」と感じること。

## 技術方針

- 言語はTypeScriptを使う
- ゲームエンジンはPhaser.jsを使う
- ビルドはViteを使う
- スタイルはCSSで管理する
- セーブはlocalStorageを使う
- PWA対応を行う
- 初期公開先はCloudflare Pagesとする

v1では、サーバー、DB、ログイン、ランキング、課金、クラウドセーブ、マルチプレイ、チャット、リアルタイム通信を実装しません。

## 画面と入力

- スマホ縦画面を最優先する
- PCでもキーボード操作できるようにする
- スマホ操作は左下の固定仮想スティック、右下の主要ボタンを基本にする
- 設定で、左下スティック/右下ボタンと右下スティック/左下ボタンを切り替えられるようにする
- 右下の大ボタンは掃除、小ボタンは回避にする
- 掃除は一度押すと開始し、もう一度押すと中断するトグル操作にする
- 「調べる」は常時ボタンにせず、宝箱、出口、素材、特殊オブジェクトに近づいた時の状況アクションとして扱う
- 素材は近づいたら自動回収する
- メニューは右上の一時停止ボタンで開く
- v1の操作は移動、トグル掃除、回避、状況アクション、清掃バトル/清掃道具アクション、メニューに絞る
- ボタン数を増やしすぎない
- タップ範囲は広めに取り、誤操作を減らす
- 主要ボタンのタップ領域は64px以上、その他のタップ領域は48px以上を目安にする
- タッチ操作中にブラウザスクロールが起きないようにする
- iPhoneのホームインジケータなど、端末のセーフエリアを考慮する
- 左右どちらの操作レイアウトでも、主要タップ領域64px以上、その他48px以上、同時押し、スクロール抑止を満たす
- 横持ちはv1対象外とし、縦画面で崩れないことを優先する

掃除操作の基本:

- 残骸に近づいて掃除ボタンを一度押すと清掃を開始する
- 対象はプレイヤーが向いている方向、または最寄りの残骸を優先する
- 掃除中は進捗リング、短い揺れ、粉じん、掃き音で手応えを出す
- もう一度掃除ボタンを押す、または掃除射程を離れると清掃を中断する
- 耐久値が低い残骸はすぐ消し、耐久値が高い残骸は複数拍かけて消す

清掃バトル/清掃道具アクションの基本:

- 現在の「攻撃」は、掃除が主役のまま危険物を追い払う/片付ける清掃バトル操作として扱う
- 文言は「攻撃」だけに寄せず、「追い払い」「片付け」「払う」も併用する
- 危険物は討伐対象ではなく、清掃作業を邪魔するものとして扱う
- 攻撃力強化は危険物を追い払う速さに影響させる
- 武器生産は危険物への対処力を伸ばす
- スタミナ強化は掃除、回避、清掃バトル行動の継続力に影響させる
- ボタン数が増えすぎる場合は、状況依存操作または掃除操作への統合を優先する
- 報酬表現は「倒した」ではなく「片付けたら素材が残った」印象に寄せる

PC操作の基本:

- `W / A / S / D`: 移動
- `方向キー`: 移動
- `Space`: 掃除
- `Shift`: 回避
- `E`: 状況アクション
- `J`: 清掃バトル/清掃道具アクション
- `Esc`: メニュー

## データ方針

- アイテム、ダンジョン、階層、生成シード、装備、生産レシピ、強化、図鑑、依頼データはTypeScriptのデータ定義として管理する
- セーブデータには `version` を含める
- セーブデータの読み込み失敗時は初期データに戻せるようにする
- 将来のDB化を見越して、保存形式は単純なJSONにする
- v1ではlocalStorageに収まる単純な進行データを優先する
- セーブには出撃回数、階層進行、敵強化状態、装備、生産状況、操作レイアウト設定を含める

## ファイル編集ルール

- 文字コードはUTF-8を使う
- 日本語Markdownは文字化けしないように確認する
- 既存のユーザー変更を不用意に戻さない
- コードや構成を変更するときは、目的に関係する範囲だけを編集する
- 旧診断サイト由来の広告、問い合わせ、AdSense、ASP運用はゲームv1には持ち込まない
- `Skills.md` のファイル名は維持する

## 確認コマンド

実装後は、プロジェクト構成に応じて次を確認します。

```powershell
npm install
npm run dev
npm run build
```

テストが追加された場合は、該当するテストコマンドも実行します。

ドキュメントだけを変更した場合は、少なくとも次を確認します。

```powershell
git diff -- README.md PLAN.md DESIGN.md BUGS.md AGENTS.md Skills.md
```
