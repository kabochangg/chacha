# Skills.md

## このプロジェクトで使う作業スキル

このファイルは、実装時に意識する専門領域と確認観点をまとめるメモです。Codexのシステムスキル定義ではなく、プロジェクト内の作業指針として扱います。

## 1. スマホ操作設計

- 縦画面を最優先にする
- 左下スティック/右下ボタンを基本にする
- 左右反転設定を必ず確認する
- 主要タップ領域64px以上、その他48px以上を守る
- `touch-action: none` などで操作中のスクロールを抑止する
- セーフエリアにUIがかからないようにする

## 2. 掃除アクション設計

- 掃除はトグル操作にする
- 対象は向いている方向、または最寄りの残骸を優先する
- 進捗リング、揺れ、粉じん、掃き音で手応えを作る
- 範囲外へ出たら中断する
- 中断した残骸は途中進捗を少し残す
- 耐久値の違いを掃除時間と清掃率増加量に反映する

## 3. 清掃道具アクション設計

- 「攻撃」ではなく「追い払い」「片付け」「払う」に寄せる
- 横薙ぎの弧を基本エフェクトにする
- 危険物を倒す表現より、作業の邪魔をどかす表現にする
- 攻撃力強化は危険物を追い払う速さとして扱う
- 武器生産は危険物への対処力を伸ばす導線として扱う

## 4. ダンジョン生成

- 「はじまりの地下道」をB1F〜B5Fで構成する
- 出撃ごとにマップ、開始位置、残骸、危険物、出口候補を変える
- 出口は条件達成後、到達可能な候補から選ぶ
- 散乱物は通行判定を持ち、掃除するまで通れない
- 1回1〜3分で終わる広さに収める

## 5. 危険物AI

- 各フロア2〜5体にする
- 内部的な視線判定を持たせる
- 発見時のみ `!` を出して追跡する
- 視線扇形の枠や範囲エフェクトは表示しない
- 壁を貫通しない
- 壁や障害物に引っかかって停止し続けない
- 深い階または出撃回数に応じて少し強くする

## 6. 報酬と成長

- バッグ初期容量は40
- 階層移動後も今回出撃中のバッグ収納数を維持する
- 素材は自動回収する
- リザルトで換金か倉庫収納を選べるようにする
- 拠点倉庫で売る/保管するを整理できるようにする
- ほうき、バッグ、攻撃力、スタミナ強化を優先する
- ほうき生産、武器生産を素材消費の目的にする
