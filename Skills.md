# Skills

## このプロジェクトで必要なスキル

「勇者のあとしまつ / ダンジョン清掃員の生活」をv1公開版として完成させるために必要な開発スキルをまとめます。

最初から大きなRPGを作り込まず、MVPの短い清掃ループを1〜3分の清掃バトル寄り短期探索へ磨きながら、タイトル、ホーム、ダンジョン、リザルト、図鑑、持ち物、依頼、ショップをv1相当の完成度に整えます。

## Phaser.js / TypeScript

- PhaserのScene、GameObject、Physics、Camera、Inputを理解する
- `BootScene`、`TitleScene`、`BaseScene`、`DungeonScene`、`ResultScene` の役割を分ける
- TypeScriptでプレイヤー、残骸、素材、危険物、階層、装備、生産レシピ、強化、セーブデータ、図鑑データの型を定義する
- ゲームロジックと表示処理を混ぜすぎない
- 毎フレームの重い処理を避け、スマホでも軽く動かす
- 図形描画、生成テクスチャ、Containerを使い、軽量なドット絵風表現を作る

## 2Dゲーム設計

- 見下ろし型の小さなダンジョンマップを作る
- 壁、床、出口、残骸、素材、危険物、宝箱を配置する
- B1F〜B5Fの短い階層構造を設計する
- 出撃ごとにマップ配置とスタート位置をある程度ランダム化する
- プレイヤー移動、当たり判定、カメラ追従を調整する
- 危険物は主役ではなく、作業を邪魔する要素として扱う
- ほうき/武器の追い払いは、掃除が主役の清掃バトル補助アクションとして設計する
- 各フロアに2〜5体のモンスターをランダム配置する
- ダンジョンに行くごと、または深い階へ行くごとにモンスターを少し強くする
- 1回1〜3分で終わるテンポを保つ
- 「もう一回だけダンジョンに行きたい」と感じる報酬導線を作る

## ランダム生成 / 階層設計

- B1F〜B5Fの各階に、短い探索と帰還判断を作る
- スタート位置、残骸、素材、危険物、宝箱、出口の候補点を管理する
- ランダム性は毎回の新鮮さを作る範囲に留め、スマホ操作で詰む配置を避ける
- 生成シードや出撃回数を保存/再現できる形にしておく
- 1〜3分で完了するよう、階層数、部屋サイズ、移動距離、清掃量を調整する

## 敵AI / 視線判定

- モンスターの通常巡回、視線扇形、発見、追跡、見失いを設計する
- 視線扇形と `!` で、プレイヤーが見つかった理由を視覚的に伝える
- 視線判定は最初は単純な距離と角度でよいが、将来の壁遮蔽を見越す
- 追跡速度や視線距離は、階層や出撃回数で少しずつ強くする
- 敵AIが強すぎて掃除テンポを壊さないように調整する

## スマホUI / 入力設計

- スマホ縦画面を最優先にする
- 左下に固定仮想スティックを置く
- 右下に掃除の大ボタンと回避の小ボタンを置く
- 設定から右下スティック/左下アクションへ操作UIを反転できるようにする
- 左右反転設定をlocalStorageに保存する
- 状況アクションは常時ボタンにせず、必要な時だけ掃除ボタンまたは近接表示を切り替える
- 素材は近づいたら自動回収する
- タップ領域を広めに取り、押し間違いを減らす
- 主要ボタンは64px以上、その他は48px以上を目安にする
- タッチ操作中のブラウザスクロールを防ぐ
- セーフエリアを考慮し、主要操作を画面端へ寄せすぎない
- 左右どちらの配置でも同時押し、スクロール抑止、タップ範囲を確認する
- PCではWASD、方向キー、Space、Shift、E、J、Escを使えるようにする

## 掃除アクション設計

- 掃除は長押し操作にする
- 長押し中は進捗リング、短い揺れ、粉じん、掃き音で手応えを出す
- 長押しを離したら掃除を中断する
- 未完了の掃除ゲージは少しずつ戻す
- 残骸の耐久値が0になったら素材をドロップする
- 耐久値が低い残骸はすぐ消し、耐久値が高い残骸は複数拍かけて消す
- 対象選択は、プレイヤーが向いている方向または最寄りの残骸を優先する
- 掃除完了時は、見た目、音、素材ポップで気持ちよく片付いた感触を出す

## 清掃バトル / 清掃道具アクション設計

- 既存の攻撃表記は、追い払い、片付け、払うなどの清掃文脈を残しつつ、攻撃力や武器の成長要素と接続する
- 主役は掃除と素材回収のままにし、危険物対処は短い清掃バトルとして扱う
- スマホでボタン数が増えすぎる場合は、状況依存操作や掃除操作への統合を検討する
- 危険物からのドロップも「片付けたら素材が残った」印象にする
- 攻撃力強化は危険物を追い払う速さに影響させる
- 武器生産は危険物への対処力を伸ばす
- スタミナ強化は掃除、回避、清掃バトル行動の継続力に影響させる

## UIアート / 演出

- 添付画像の方向性に合わせ、暗い石畳、暖色のランタン光、黒革、木目、金縁UIを基調にする
- タイトル/ホーム画面は上部リソースバー、中央メニュー、今日の清掃メモ、下部タブを持たせる
- ダンジョン画面は上部HUD、右上ミニマップ風パネル、B1F〜B5F表示、左右反転可能なスティック/アクションボタンを持たせる
- モンスターの向きは視線扇形で見せ、発見時は `!` と追跡演出を入れる
- 図鑑は羊皮紙風の背景に、モンスター、素材、宝、プレイヤー成長を見やすく並べる
- 背景はタイルベースで軽量に作る
- 光、影、粉じん、短い揺れは控えめに使う
- 画像や音声はファイルサイズを抑える
- 仮素材でも、用途と違いが分かる名前にする

## 図鑑 / 持ち物 / 依頼 / ショップ

- 図鑑では、モンスター、素材、宝、プレイヤー成長の収集状況を表示する
- 持ち物では、素材の個数、説明、売却価値、バッグ容量を表示する
- 依頼では、次のプレイで狙う短い清掃目標を提示する
- ショップでは、ほうき強化、バッグ強化、攻撃力強化、スタミナ強化、ほうき生産、武器生産を分かりやすく扱う
- 設定では、操作UI左右反転、音量、ヘルプを扱う
- v1では複雑なクラフト、分岐クエスト、長期イベントは扱わない

## localStorageセーブ

- 所持金、所持素材、ほうきレベル、バッグレベル、攻撃力、スタミナ、装備、生産状況、進行状況、設定、図鑑収集状況を保存する
- 階層、生成シード、出撃回数、敵強化状態を保存対象として設計する
- 操作レイアウト設定は `leftStickRightButtons` と `rightStickLeftButtons` のように明確な値で保存する
- セーブデータには `version` を含める
- 壊れたセーブデータを読んでもゲームが停止しないようにする
- 保存形式は将来のDB化を見越して単純なJSONにする

## PWA / Webアプリ化

- `manifest.json` を整える
- アイコン、テーマカラー、画面向き、表示モードを設定する
- Service Workerで基本ファイルをキャッシュする
- ホーム画面追加とオフライン起動の基本動作を確認する
- スマホブラウザの音声自動再生制限を考慮する
- PWAキャッシュ更新時の確認手順を持つ

## Cloudflare Pages

- Viteの静的ビルド成果物をCloudflare Pagesで公開できるようにする
- v1ではサーバー、DB、認証、ランキングを使わない
- 将来サーバーが必要になった場合は、Cloudflare Workers、Supabase、Firebase、Cloudflare D1などを検討する
- 公開前にスマホ実機、PC、ブラウザサイズ変更を確認する

## ゲームバランス

- ほうき強化は清掃速度に直結させる
- バッグ強化は持ち帰れる素材数に直結させる
- 攻撃力強化は危険物を追い払う速さに直結させる
- スタミナ強化は掃除、回避、清掃バトルの継続力に直結させる
- ほうき生産は清掃効率、武器生産は危険物対処力を伸ばす
- 報酬は分かりやすく、次の強化目標が見えるようにする
- 危険物は強くしすぎず、作業中の緊張感を作る程度にする
- B1F〜B5Fを1〜3分で遊べるように、階層ごとの密度を調整する
- 依頼と図鑑は、次のプレイ理由を作るために使う
- 「もう一回だけダンジョンに行きたい」と感じる短いループを優先する

## QA / テスト観点

- タイトルからホーム、ダンジョン、リザルトへ進めるか
- 1回1〜3分で出撃が終わるか
- B1F〜B5Fを進行できるか
- 出撃ごとにマップ配置とスタート位置がある程度変わるか
- スマホで移動と掃除がしやすいか
- 操作UIを左右反転しても移動とアクションがしやすいか
- 移動しながら掃除ボタンを押し続けられるか
- 掃除、回避、状況アクションを押し間違えにくいか
- PCキーボードで操作できるか
- 壁をすり抜けないか
- 残骸を掃除して素材を拾えるか
- 危険物を避ける、または清掃道具で追い払えるか
- モンスターの視線扇形、発見 `!`、追跡が分かりやすいか
- 出口から帰還できるか
- 所持金と素材が増えるか
- 強化できるか
- 素材でほうきや武器を生産できるか
- 図鑑、持ち物、依頼、ショップが破綻なく表示されるか
- セーブと続きから開始が動くか
- PWAとしてホーム画面に追加できるか
- ビルド後もCloudflare Pagesで公開できるか
