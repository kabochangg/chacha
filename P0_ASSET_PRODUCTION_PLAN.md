# P0アセット制作計画

## 目的

v1公開前に最低限必要な見た目要素をそろえ、タイトルからB5F帰還まで遊べる状態にします。P0では「完成絵」よりも、役割が分かり、軽く動き、スマホで読めることを優先します。

## P0で必ず作るもの

- プレイヤー清掃員
- ほうき
- バッグ
- 石床、壁、影
- 小/中/大がれき
- 木材、液体、灰、壊れた宝箱
- 石材、木材、粘液素材、灰、金属片
- 危険物4種
- 出口の開放前/開放後
- 掃除進捗リング
- 粉じん
- 発見 `!`
- 横薙ぎの清掃道具エフェクト
- 仮想スティック、掃除ボタン、走るボタン
- ホーム、リザルト、下部タブの最低限UI
- PWAアイコン

## 制作ステップ

### Step 1: 識別用シルエット

- プレイヤー、危険物、素材、がれき、出口を単色に近い形で作る
- スマホ縦画面で見た時に判別できるか確認する
- 色だけではなく形で区別できるようにする

### Step 2: 掃除の手応え

- 掃除リングを出す
- ほうき方向へ粉じんを出す
- 完了時に対象を縮小し、素材をポップさせる
- 中断時にリングを戻す

### Step 3: 危険物の状態表現

- 通常巡回
- 発見 `!`
- 追跡
- 追い払い
- 接触ダメージ

視線扇形の枠や範囲エフェクトは作りません。

### Step 4: UI最低限

- タイトル
- ホーム
- ダンジョンHUD
- 操作UI
- リザルト
- 下部タブ

空タブは残さず、最低限の一覧または準備中表示を入れます。

### Step 5: PWA

- アプリアイコン
- マスク可能アイコン
- manifest確認
- Service Workerのキャッシュ名更新

## 完了条件

- プレイヤー、掃除対象、素材、危険物、出口が一目で区別できる
- 掃除中、掃除完了、素材回収が見て分かる
- 危険物の発見 `!` と追跡が見て分かる
- スマホ縦画面で主要UIが読める
- `npm run build` が通る
