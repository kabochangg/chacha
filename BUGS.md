# v1前の課題管理メモ

このファイルは、v1公開前に確認すべき不具合、仕様ズレ、改善候補、検証済み項目を整理するメモです。

## 重点確認

- 素材回収が複数回カウントされないこと
- 危険物接触でHPまたはスタミナが減ること
- タッチ操作中にブラウザスクロールが起きないこと
- PWAキャッシュが古いファイルを残さないこと
- 操作UI左右反転後も移動とアクションを同時に使えること
- B1F〜B5Fを1〜3分で進めるテンポになっていること
- 出口が到達不能な位置に出ないこと

## 仕様ズレとして扱うもの

- 旧サイト由来の広告、問い合わせ、AdSense、ASP運用の記述や導線が残っている
- 「敵を倒す」「討伐する」印象が強すぎる
- 視線扇形の枠や範囲エフェクトが表示される
- 図鑑、持ち物、倉庫、依頼、ショップが空タブや壊れた導線になる
- バッグ容量の初期値が40ではない
- 階層移動後に今回出撃中のバッグ収納数が消える
- 散乱物を掃除しなくても通れてしまう
- 危険物が壁を貫通する、または障害物に引っかかって停止し続ける

## 改善候補

- 一時停止メニューで今回出撃中の素材、バッグ容量、清掃率、HPまたはスタミナ、所持金を確認できるようにする
- リザルトで素材の換金と倉庫収納の判断を分かりやすくする
- 倉庫で素材を売る、保管する、生産に使う流れを整理する
- ダンジョン右上にミニマップ風パネルと清掃進捗を出す
- 図鑑をモンスター、素材、宝、プレイヤー成長の一覧として独立させる
- 清掃道具アクションの横薙ぎ演出を短く気持ちよくする
- 小/中/大がれきで清掃率の増加量を変える

## 検証済みとして記録する条件

- `npm install` が成功する
- `npm run dev` でローカル起動できる
- `npm run build` が成功する
- タイトルからホーム、ダンジョン、リザルトへ進める
- B1F〜B5Fを進行できる
- 各フロアに2〜5体の危険物が出る
- 危険物の発見 `!` と追跡が機能する
- 視線扇形の枠や範囲エフェクトが表示されない
- 散乱物が掃除するまで通行障害になる
- 素材を1つ拾った時、素材数が1だけ増える
- リザルトで素材を換金または倉庫収納できる
- 拠点倉庫で素材を売るか保管するか整理できる
- ほうき、バッグ、攻撃力、スタミナを強化できる
- ほうきや武器を生産できる
- 左右どちらの操作レイアウトでも同時押しできる
- PWA用の `manifest.json`、Service Worker、アイコンがビルド成果物に含まれる

## 公開後の改善優先順位

1. 実機スマホで押しづらいボタン位置とセーフエリアを直す
2. 掃除完了、素材回収、危険物の追い払い演出を磨く
3. 報酬量、強化費用、生産素材数を調整する
4. B1F〜B5Fの移動距離と清掃量を見直す
5. 図鑑画面を収集済み/未収集が分かる画面として育てる
