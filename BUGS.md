# 公開前に直すこと

## 最優先

- Markdownとゲーム内文言の文字化けを直す
- 添付画像の方向性に合わせて、メニュー画面とプレイ画面のUIを再整理する
- スマホ縦画面で操作領域が重ならないようにする
- 掃除ボタン、回避ボタン、状況アクションの見た目と当たり判定を一致させる
- 仮想スティック操作中にブラウザスクロールが起きないことを確認する

## 操作UI

- 掃除ボタンと回避ボタンが近すぎる場合は、見た目ではなく当たり判定も含めて調整する
- 右下の大ボタンは掃除、右下の小ボタンは回避に固定する
- 状況アクションは常時ボタンにせず、出口、宝箱、素材、特殊オブジェクトに近づいた時だけ分かるようにする
- 主要ボタンのタップ領域は64px以上、その他のタップ領域は48px以上を目安にする
- セーフエリアを考慮し、iPhoneのホームインジケータに主要操作がかぶらないようにする

## 画面幅とレイアウト

- スマホ縦画面でcanvasとゲーム表示の左右に不自然な余白が出ないようにする
- HUD、ミニマップ風パネル、操作ボタンが互いに重ならないようにする
- 文字サイズはスマホ実機で読める大きさにする
- 横持ちはMVP対象外だが、縦画面で崩れないことを優先する

## オブジェクトの分かりやすさ

- 残骸、素材、危険物、出口の役割が見た目で分かるようにする
- ダメージ源は、接触前に危険だと判断できる色、形、警告演出を入れる
- 出口は光、看板、矢印、メッセージなどで位置と役割を明確にする
- 素材回収時は、通知と音で「拾った」感覚を出す

## 掃除アクション

- 長押し中の進捗リングを分かりやすくする
- 残骸の短い揺れ、粉じん、掃き音を入れる
- 長押しを離したら清掃が中断されることを確認する
- 耐久値が低い残骸はすぐ消え、耐久値が高い残骸は複数拍かけて消えるようにする
- 対象選択は、プレイヤーが向いている方向または最寄りの残骸を優先する

## セーブ/PWA/公開

- `manifest.json` の名前、短い名前、テーマカラー、縦画面指定を日本語タイトルに合わせる
- Service Workerのキャッシュ名を公開前に更新する
- localStorageの読み込み失敗時に初期データへ戻ることを確認する
- `npm run build` が通ることを確認する
- Cloudflare Pagesで公開できることを確認する

## 確認メモ

公開前に最低限確認すること:

- タイトルから開始できる
- 拠点からダンジョンに入れる
- 移動できる
- 掃除できる
- 素材を拾える
- 出口から帰還できる
- リザルトで所持金が増える
- ほうきまたはバッグを強化できる
- ブラウザを閉じても続きから遊べる
- スマホで移動しながら掃除ボタンを押し続けられる
