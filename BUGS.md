# v1前の課題管理メモ

このファイルは、v1公開前に確認すべき不具合、仕様ズレ、改善候補、検証済み項目を整理するためのメモです。

## 不具合

- 素材回収が複数回カウントされないことを継続確認する。
  - 回収開始時点で `collected` を立て、物理判定を無効化する方針。
  - 確認: 素材を1つ拾った時、素材数が1だけ増える。

- 危険物接触でHPが減らないケースがないか確認する。
  - 矩形判定だけに頼らず、中心距離でも接触判定する方針。
  - 連続ヒットしすぎないよう、短い無敵時間は維持する。
  - 確認: 危険物に触れるとHPが減る。

- タッチ操作中にブラウザスクロールが起きないか確認する。
  - 仮想スティック、掃除ボタン、回避ボタンを同時操作した時も見る。

- PWAキャッシュが古いファイルを残さないか確認する。
  - 公開前にキャッシュ名とService Worker更新手順を確認する。

## 仕様ズレ

- 旧仕様の「1フロア」「旧プレイ時間」「固定左右操作」「非戦闘寄り表現」が残っていないか確認する。
  - v1ではB1F〜B5F、1回1〜3分、左右反転操作、清掃バトル寄りの危険物対処を基準にする。

- 現在の「攻撃」表記とボタンは、v1方針では正式戦闘ではない。
  - 「追い払い」「片付け」「払う」などの清掃文脈を残しつつ、攻撃力や武器の成長要素と接続する。
  - スマホでボタン数が増えすぎる場合は、状況依存操作または掃除操作への統合を検討する。
  - PCの `J` キーは、当面「清掃道具アクション」として扱う。

- モンスターを倒す報酬に見える表現は避ける。
  - 「倒した」より「追い払った」「片付いた」「素材が残った」という文言を優先する。

- 図鑑、持ち物、依頼、ショップが空のタブに見えないようにする。
  - v1では最低限の一覧、説明、準備中メッセージを用意する。

## 改善候補

- 一時停止メニューから、今回ダンジョン内で持っている素材、バッグ容量、清掃率、HP、所持金を確認できるようにする。
- ダンジョン画面右上にミニマップ風パネルと `広間の清掃 4/5` のような進捗表示を置く。
- ダンジョンをB1F〜B5F構成にし、B5Fを最奥地として扱う。
- 出撃ごとにマップ配置とスタート位置をある程度ランダム化する。
- 各フロアのモンスター数を2〜5体の範囲でランダム化する。
- ダンジョンに行くごと、または深い階へ行くごとにモンスターを少し強くする。
- モンスターの向きが分かる視線扇形を出す。
- モンスターがプレイヤーに気づいたら `!` を出して追いかける。
- 設定から操作UIを左下スティック/右下アクションと右下スティック/左下アクションで反転できるようにする。
- タイトル/ホーム画面に上部リソースバー、今日の清掃メモ、下部タブを追加する。
- 図鑑にモンスター、素材、宝、プレイヤー成長の見た目を整理して表示する。
- 素材ごとに形と色を変え、スマホでも見分けやすくする。
- 危険物の見た目を赤い四角から、警告色、影、揺れ、`!` を持つデザインにする。
- 追い払い時の演出を、斬撃ではなくほうきの弧、粉じん、軽い後退に寄せる。
- 素材でほうきや武器を生産できるUIを追加する。
- 攻撃力、スタミナ、ほうき、バッグの強化導線を分かりやすくする。
- がれきの大きさごとに清掃率の増加量を変える。
  - 小さいがれき: 5%
  - 中くらいのがれき: 10%
  - 大きいがれき: 15%
- 掃除途中で掃除ボタンを離すと、未完了がれきの個別掃除ゲージが少しずつ戻る。

## 検証済みとして扱う条件

- `npm run build` が成功する。
- タイトルからホーム、ダンジョン、リザルトへ進める。
- 1回1〜3分で出撃が終わる。
- B1F〜B5Fを進行でき、B5Fが最奥地として扱われる。
- 出撃ごとにマップ配置とスタート位置がある程度変わる。
- 各フロアに2〜5体のモンスターが出る。
- モンスターの視線扇形、発見 `!`、追跡が分かりやすい。
- 素材を1つ拾った時、素材数が1だけ増える。
- メニューで今回の素材数が確認できる。
- 危険物接触でHPが減る。
- 清掃道具アクションで危険物を追い払える。
- 素材でほうきや武器を生産できる。
- 小/中/大がれきの清掃完了で清掃率が5/10/15%増える。
- 掃除を中断すると、掃除中だったがれきの進捗が徐々に戻る。
- 右下操作ボタンと左下仮想スティックを同時に使っても、画面スクロールが起きない。
- 操作UIを左右反転した後も、移動とアクションを同時に使える。
