# v1完成までの計画

## 目的

「勇者のあとしまつ / ダンジョン清掃員の生活」を、Cloudflare Pagesで公開できるv1へ仕上げるための作業計画です。長期運用版ではなく、まず人に遊んでもらえる短時間清掃ゲームとして完成させます。

## Phase 1: 仕様と文書の整理

- README、PLAN、DESIGN、BUGS、AGENTS、Skillsをv1方針に統一する
- 旧サイト由来の広告、問い合わせ、AdSense、ASP運用の記述を残さない
- 「攻撃」表現を清掃道具アクション、追い払い、片付けへ寄せる
- v1対象外を明記する

完了条件:

- MarkdownがUTF-8の日本語として読める
- 実装者が優先順位と受け入れ基準を判断できる

## Phase 2: 操作と掃除の手触り

- 仮想スティックとアクションボタンをスマホ縦画面向けに調整する
- 左右反転設定をlocalStorageへ保存する
- 掃除をトグル操作にする
- 掃除中の進捗リング、揺れ、粉じん、掃き音を強化する
- 掃除範囲を離れたら中断する
- 散乱物を掃除するまで通れない障害物として扱う
- 素材を近づいたら自動回収する

完了条件:

- 移動しながら掃除を継続できる
- スマホで誤操作が少ない
- 主要タップ領域64px以上、その他48px以上を満たす
- タッチ操作中にブラウザスクロールが起きない

## Phase 3: ダンジョン進行

- 「はじまりの地下道」をB1F〜B5F構成にする
- B5Fを最奥地として扱う
- 出撃ごとにマップ配置とスタート位置をランダム化する
- 条件達成後の出口位置を到達可能候補からランダム化する
- 1回1〜3分で終わる広さと清掃量に調整する

完了条件:

- B1F〜B5Fを進行できる
- 出撃ごとに配置差がある
- 出口が到達不能な場所に出ない

## Phase 4: 危険物AIと清掃道具アクション

- 各フロアの危険物数を2〜5体でランダム化する
- 深い階または出撃回数に応じて少し強くする
- 内部的な視線判定、発見 `!`、追跡を実装する
- 視線扇形の枠や範囲エフェクトは表示しない
- 壁や障害物に引っかかって停止し続けないようにする
- 清掃道具アクションは横薙ぎの弧を基本にする

完了条件:

- 危険物が発見時に分かりやすく反応する
- 危険物が壁を貫通しない
- 危険物が永久に詰まり続けない
- 表現が討伐より「追い払い」「片付け」に寄っている

## Phase 5: 報酬、倉庫、強化、生産

- バッグ初期容量を40にする
- 階層移動後も今回出撃中のバッグ収納数を維持する
- リザルトで素材を換金するか倉庫へ収納するか選べるようにする
- 拠点倉庫で素材を売るか保管するか整理できるようにする
- ほうき、バッグ、攻撃力、スタミナの強化を入れる
- ほうき生産、武器生産を入れる

完了条件:

- 回収した素材の使い道が分かる
- 所持金が増える
- 強化後に次の出撃が少し楽になる
- 素材を売るだけでなく生産にも使える

## Phase 6: 画面と最低限の導線

- タイトル、ホーム、ダンジョン、リザルトを完成形へ近づける
- 図鑑、持ち物、倉庫、マップ、依頼、ショップを最低限実装する
- 空タブや壊れた導線を残さない
- 未実装要素は短い準備中表示に留める

完了条件:

- ホーム下部タブが意味を持つ
- 図鑑で危険物、素材、宝、成長情報を見られる
- 依頼で次の清掃目標が分かる
- ショップで強化導線が分かる

## Phase 7: セーブ、PWA、公開

- save dataに `version` を含める
- 読み込み失敗時は初期データへ戻せるようにする
- PWAのmanifest、アイコン、Service Workerを整える
- Cloudflare Pages向けのビルド設定を確認する
- `npm install`、`npm run dev`、`npm run build` を確認する

完了条件:

- ブラウザを閉じても続きから遊べる
- ホーム画面へ追加できる
- `npm run build` が成功する
- Cloudflare Pagesで公開できる

## 公開前チェック

- スマホ実機で縦画面、同時押し、セーフエリア、スクロール抑止を確認する
- PCでキーボード操作を確認する
- B1F〜B5Fを通して1〜3分のテンポに収まるか確認する
- 報酬量、強化費用、生産素材数を調整する
- 公開URLでタイトル、ホーム、ダンジョン、リザルト、PWA登録を確認する
