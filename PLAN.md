# 勇者のあとしまつ v1完成までの段階計画

## 目的

Webアプリゲーム「勇者のあとしまつ / ダンジョン清掃員の生活」を、MVPからv1公開版へ磨き込む。

v1の完成形は、長期運用版ではなく、Cloudflare Pagesで公開できる最初の完成版です。1〜3分の清掃バトル寄りの短期探索、スマホ操作の快適さ、報酬と強化/生産の分かりやすさを優先します。

## v1の完成イメージ

添付されたタイトル、ホーム、プレイ画面、モンスター図鑑、プレイヤー図鑑、素材・宝図鑑を、v1の方向性として採用します。

- 暗い石畳、ランタンの暖色光、苔、骨、壊れた宝箱、スライム跡で「勇者が去った後」の空気を出す
- UIは黒革、木目、金縁、羊皮紙風パネルを基調にする
- タイトル/ホーム画面は、上部リソースバー、中央メニュー、今日の清掃メモ、下部タブを持つ
- ダンジョン画面は、上部HUD、右上ミニマップ風パネル、B1F〜B5F表示、清掃進捗、仮想スティック、アクションボタンを持つ
- 図鑑は、モンスター、素材、宝、プレイヤー成長が見られる収集画面にする
- 設定で左下スティック/右下アクションと右下スティック/左下アクションを切り替えられる
- 添付画像そのものは素材として直貼りせず、Phaser描画と軽量なドット絵風表現で雰囲気を寄せる

## Phase 1: 文書・仕様整理

目的: 実装判断の基準をv1公開版に揃える。

- `README.md`、`PLAN.md`、`DESIGN.md`、`BUGS.md`、`Skills.md`、`AGENTS.md` をUTF-8の日本語で整理する
- MVPの説明を、v1公開版へ向けたロードマップに更新する
- 「攻撃」は清掃バトル寄りの補助操作として、掃除が主役のまま攻撃力/武器/危険物対処を扱えるように再定義する
- 旧「副業タイプ診断」由来の広告、問い合わせ、AdSense、ASP運用の要素を公開物に残さない
- v1対象外を明記する: サーバー、DB、ログイン、ランキング、課金、クラウドセーブ、マルチプレイ、チャット、リアルタイム通信

完了条件:

- Markdownが文字化けしていない
- v1完成までの作業順と受け入れ基準が明確になっている
- 実装者が「何を優先し、何をやらないか」を判断できる

## Phase 2: 操作と清掃バトル体験の磨き込み

目的: スマホで気持ちよく遊べる清掃アクションと、短く分かりやすい危険物対処を作る。

- 左下仮想スティックと右下操作ボタンを、スマホ縦画面で押しやすい位置とサイズに調整する
- 設定から、左下スティック/右下アクションと右下スティック/左下アクションを反転できるようにする
- 操作レイアウト設定はlocalStorageのsettingsに保存する
- 掃除は長押し操作にし、移動しながら押し続けられることを保証する
- 掃除中の進捗リング、短い揺れ、粉じん、掃き音を強化する
- 長押しを離した時は掃除を中断し、未完了がれきのゲージが少しずつ戻る
- 素材は近づいたら自動回収する
- 宝箱、出口、特殊オブジェクトだけ状況アクションを出す
- ほうきの追い払いは、掃除が主役の清掃バトル操作として見た目と文言を整える
- 攻撃力強化は危険物を追い払う速さに影響させる
- スタミナ強化は掃除、回避、清掃バトル行動の継続力に影響させる
- ボタン数が増えすぎる場合は、状況依存操作または掃除操作への統合を優先する

完了条件:

- プレイヤーを操作できる
- 移動しながら掃除ボタンを長押しできる
- 残骸を掃除できる
- 素材を拾える
- 危険物を避ける、または追い払える
- 左右どちらの操作レイアウトでも仮想スティック操作中にブラウザスクロールが起きない

## Phase 3: UI/画面構成とグラフィックの完成形寄せ

目的: 添付イメージの方向性に沿って、タイトルからダンジョンまでの見た目、導線、操作しやすさ、報酬の分かりやすさを整える。グラフィック制作は「見た目だけ」ではなく、スマホで読みやすく、掃除が気持ちよく、もう一回行きたくなる手触りを上げる作業として扱う。

- タイトル画面を、暗いダンジョン背景、ゲームロゴ、短いキャッチコピー、`はじめる`、`つづきから`、`設定`、`図鑑`、`ヘルプ` の構成にする
- ホーム画面は、上部リソースバー、今日の清掃メモ、主要メニュー、下部タブを持つ拠点にする
- 下部タブは `ホーム`、`持ち物`、`マップ`、`依頼`、`ショップ` を表示する
- ダンジョン画面は、上部HUD、右上ミニマップ風パネル、B1F〜B5F表示、清掃進捗、仮想スティック、操作ボタンを整理する
- リザルト画面は、清掃率、素材、売却額、所持金、強化までの距離、作業時間を見やすくする
- タイトル背景、ロゴ周り、開始ボタン、拠点パネル、下部タブ、HUD、清掃リング、粉じん、視線扇形、出口光、報酬演出を制作単位として順に磨く
- プレイヤー、残骸、素材、危険物、出口は、色だけでなく形、光、動きで役割が分かるようにする
- 外部画像の直貼りではなく、Phaser図形描画、生成テクスチャ、軽量なドット絵風パーツで画面密度を上げる
- 未実装機能は空の詳細画面にせず、v1で必要な最低限の情報と短い準備中メッセージに留める

完了条件:

- タイトルからホームへ進める
- ホームからダンジョンへ入れる
- 強化に必要な金額と現在の所持金が分かる
- ダンジョン内で階層、清掃進捗、HP/スタミナ、素材容量が分かる
- スマホ縦画面で主要ボタンが押しやすい
- 設定で操作UIを反転しても表示と当たり判定が崩れない
- 初見でも、掃除対象、回収素材、危険物、出口、報酬導線が見た目で分かる
- リザルトの「もう一回行く」が自然に押したくなる位置と演出になっている

## Phase 4: 図鑑・持ち物・依頼・ショップの最低限実装

目的: v1公開版として、下部タブの期待に応える最低限の機能を入れる。

- 図鑑では、モンスター、素材、宝、プレイヤー成長の収集状況を見られるようにする
- 持ち物では、所持素材、素材説明、売却価値、バッグ容量を確認できるようにする
- マップでは、現在のダンジョン「はじまりの地下道」と進行状況を表示する
- 依頼では、短い清掃目標を提示し、次にやることが分かるようにする
- ショップでは、ほうき強化、バッグ強化、攻撃力強化、スタミナ強化、ほうき生産、武器生産を分かりやすく配置する
- 素材は売却だけでなく、ほうきや武器の生産/強化にも使う
- 設定画面では操作UI左右反転、音量、ヘルプを扱う
- v1では複雑なクエスト分岐は入れず、生産は次の清掃理由を作る範囲に留める

完了条件:

- 下部タブを押しても空白や破綻した画面にならない
- 図鑑でモンスター/素材/宝の情報が見られる
- 持ち物とショップで強化の意味が分かる
- 素材でほうきや武器を生産できる理由が分かる
- 操作UI反転設定を変更できる
- 依頼が次のプレイ目標として機能する

## Phase 5: 階層ダンジョン、敵AI、報酬バランス調整

目的: 1回1〜3分のプレイとして、B1F〜B5Fの短期探索を成立させる。

- v1では「はじまりの地下道」をB1F〜B5F構成にし、B5Fを最奥地にする
- 各出撃ごとにマップ配置とスタート位置をある程度ランダム化する
- 各フロアのモンスター数は2〜5体の範囲でランダムにする
- ダンジョンに行くごと、または深い階へ行くごとにモンスターを少し強くする
- モンスターは通常巡回、視線扇形、発見時の `!`、追跡状態を持つ
- 清掃率80%以上、または出口到達で帰還できるようにする
- 残骸、素材、危険物、宝箱、出口の配置を、スマホ操作でも詰まないように調整する
- リザルトで清掃率、回収素材、売却見込み、獲得金額、被ダメージ、作業時間、ランクを表示する
- 所持金を増やす
- ほうき強化/生産で清掃速度や清掃範囲を上げる
- バッグ強化で持ち帰れる素材数を増やす
- 攻撃力強化と武器生産で危険物への対処力を上げる
- スタミナ強化で掃除、回避、清掃バトル行動の継続力を上げる
- 強化後、次のプレイが少し楽になったことを感じられるようにする

完了条件:

- 1〜3分で1回のプレイが終わる
- B1F〜B5Fを進行できる
- 出撃ごとにマップ配置とスタート位置の変化がある
- モンスター数が2〜5体の範囲で変化する
- モンスターの視線扇形、発見 `!`、追跡が機能する
- 報酬を得た感覚がある
- 所持金が増える
- ほうき、バッグ、攻撃力、スタミナを強化できる
- 素材でほうきや武器を生産できる
- もう一度ダンジョンに行く理由がある

## Phase 6: セーブ/PWA/Cloudflare Pages公開

目的: Webアプリとして公開できる状態にする。

- localStorageに `version` 付きの単純なJSONで保存する
- 読み込み失敗時は初期データへ戻す
- ブラウザを閉じても続きから遊べるようにする
- `manifest.json` のタイトル、短い名前、テーマカラー、縦画面指定、アイコンを整える
- Service Workerで基本ファイルをキャッシュする
- Cloudflare Pagesで静的ビルドを公開できる構成にする
- 404、headers、redirects、robots、sitemapをゲーム公開用に整える

完了条件:

- `npm run build` が通る
- ブラウザ再起動後も続きから遊べる
- ホーム画面に追加できる
- オフライン起動の基本動作を確認できる
- Cloudflare Pagesで公開できる

## Phase 7: 実機確認と公開後改善

目的: v1公開前後の品質を安定させる。

- スマホ実機で、縦画面、タッチ操作、セーフエリア、スクロール抑止を確認する
- PCで、キーボード操作とブラウザサイズ変更を確認する
- PWAインストール後の起動とキャッシュ更新を確認する
- 公開後に、操作しづらい場所、見づらいUI、報酬が弱い場所を優先して直す
- 新ダンジョンや長期機能は、v1の清掃ループが気持ちよくなってから検討する

完了条件:

- スマホ実機で主要操作が破綻しない
- Cloudflare Pages上で主要導線を遊べる
- 公開後改善の優先順位が見えている

## リスクと対策

- 画像のような高密度アートを完全再現しようとすると重くなるため、v1では軽量なドット絵風表現に絞る
- 操作が複雑になるとスマホで遊びにくくなるため、移動、長押し掃除、回避、状況アクション、メニューを主軸にし、左右反転にも対応する
- 清掃バトルが強くなりすぎるとゲームの軸がぶれるため、文言と演出は「掃除のために危険物を片付ける」に寄せる
- 5階構成が長くなりすぎるとテンポが落ちるため、1回1〜3分で到達または撤退判断できる密度にする
- UIの見た目と当たり判定がずれると誤操作が増えるため、表示オブジェクトとインタラクティブ領域を同じ座標設計にする
- 旧サイトの不要ファイルや文言が混ざると公開品質が落ちるため、ゲームv1に不要な要素は持ち込まない
- PWAキャッシュが古いファイルを残す可能性があるため、公開前にキャッシュ名と更新手順を確認する
