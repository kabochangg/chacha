# 勇者のあとしまつ 公開までの計画

## 目的

Webアプリゲーム「勇者のあとしまつ / ダンジョン清掃員の生活」を、スマホ縦画面で遊べるMVPとしてCloudflare Pagesへ公開する。

最初の成功条件は、プレイヤーが「もう一回だけダンジョンに行きたい」と感じること。豪華さよりも、短い清掃ループの気持ちよさ、回収と報酬の分かりやすさ、スマホ操作の快適さを優先する。

## 目標イメージ

添付されたメニュー画面とプレイ画面を、MVPのビジュアル方向性として採用する。

- 暗い石畳のダンジョン、ランタンの暖色光、苔、残骸、宝箱、スライム跡で「勇者が去った後」の空気を出す
- UIは黒革/木目/金縁を基調にする
- メニュー画面は、上部リソースバー、中央の大きなタイトル、開始/続き/設定系ボタン、下部タブを持つ
- プレイ画面は、上部HUD、右上ミニマップ風パネル、左下仮想スティック、右下アクションボタンを持つ
- 添付画像そのものは素材として直貼りせず、Phaser描画と軽量なドット絵風表現で雰囲気を寄せる

## Phase 1: 文言とドキュメント復旧

目的: 文字化けをなくし、公開までの判断基準を明確にする。

- `README.md`、`PLAN.md`、`BUGS.md`、`Skills.md`、`AGENTS.md` をUTF-8の日本語で整理する
- ゲーム内の文字化けしたラベル、説明文、アイテム名、リザルト文言を復旧する
- 旧「副業タイプ診断」由来の広告、問い合わせ、AdSense、ASP運用の要素を公開物に残さない
- MVPの対象外を明記する: サーバー、DB、ログイン、ランキング、課金、クラウドセーブ、マルチプレイ、チャット、リアルタイム通信

完了条件:

- Markdownが文字化けしていない
- ゲーム内の主要テキストが日本語で読める
- 公開までの作業順と受け入れ基準が明確になっている

## Phase 2: メニュー/拠点画面の更新

目的: 添付のメニュー画面を目標に、タイトルから拠点までの導線を整える。

- タイトル画面を、暗いダンジョン背景、ゲームロゴ、短いキャッチコピー、`はじめる`、`つづきから`、`設定`、`図鑑`、`ヘルプ` の構成にする
- 上部に所持金、素材、スタミナ風リソース、設定ボタンを表示する
- 拠点画面は「清掃員の拠点」として、ダンジョン開始、ほうき強化、バッグ強化、所持状況を分かりやすく配置する
- 下部タブは `ホーム`、`持ち物`、`マップ`、`依頼`、`ショップ` を表示する
- MVPで未実装のタブは、詳細画面を作り込みすぎず短いメッセージ表示に留める

完了条件:

- タイトルから拠点へ進める
- 拠点からダンジョンへ入れる
- 強化に必要な金額と現在の所持金が分かる
- スマホ縦画面でボタンが押しやすい

## Phase 3: プレイ画面と掃除体験の更新

目的: 添付のプレイ画面を目標に、操作しやすく気持ちいい清掃アクションを作る。

- 上部HUDにゲーム名/現在地、所持金、素材、スタミナ、ポーズボタンを表示する
- 右上にミニマップ風パネルと `広間の清掃 4/5` のような進捗表示を置く
- 左下に固定仮想スティックを置く
- 右下に大きい掃除ボタン、小さい回避ボタン、状況アクション表示を置く
- 掃除は長押しにし、進捗リング、短い揺れ、粉じん、掃き音で手応えを出す
- 素材は近づいたら自動回収する
- 宝箱、出口、素材、特殊オブジェクトに近づいた時だけ状況アクションを表示する
- 出口は見た目、看板、光、メッセージで場所と役割が分かるようにする
- ダメージ源は危険だと分かる見た目と接触リアクションを入れる

完了条件:

- プレイヤーを操作できる
- 移動しながら掃除ボタンを長押しできる
- 残骸を掃除できる
- 素材を拾える
- 出口から帰還できる
- 仮想スティック操作中に画面スクロールが起きない

## Phase 4: ゲームループ完成

目的: 1回3〜5分のプレイとして成立させる。

- 「はじまりの地下道」1つに絞って密度を上げる
- 清掃率80%以上、または出口到達で帰還できるようにする
- リザルトで清掃率、回収素材、売却見込み、獲得金額、被ダメージ、作業時間、ランクを表示する
- 所持金を増やす
- ほうき強化で清掃速度を上げる
- バッグ強化で持ち帰れる素材数を増やす
- 強化後、次のプレイが少し楽になったことを感じられるようにする

完了条件:

- 3〜5分で1回のプレイが終わる
- 報酬を得た感覚がある
- 所持金が増える
- ほうきまたはバッグを強化できる
- もう一度ダンジョンに行く理由がある

## Phase 5: セーブ/PWA/公開

目的: Webアプリとして公開できる状態にする。

- localStorageに `version` 付きの単純なJSONで保存する
- 読み込み失敗時は初期データへ戻す
- ブラウザを閉じても続きから遊べるようにする
- `manifest.json` のタイトル、短い名前、テーマカラー、縦画面指定、アイコンを整える
- Service Workerで基本ファイルをキャッシュする
- Cloudflare Pagesで静的ビルドを公開できる構成にする
- 404、headers、redirects、robots、sitemapをゲーム公開用に整える

完了条件:

- `npm run build` が通る
- ブラウザ再起動後も続きから遊べる
- ホーム画面に追加できる
- オフライン起動の基本動作を確認できる
- Cloudflare Pagesで公開できる

## 受け入れテスト

基本動作:

- タイトル画面が表示される
- `はじめる` で拠点に移動する
- `つづきから` で保存済み状態から再開する
- 拠点からダンジョンに入れる
- プレイヤーが移動できる
- 壁をすり抜けない
- 掃除ボタン長押しで残骸を消せる
- 素材を自動回収できる
- 出口から帰還できる
- リザルトが表示される
- 所持金が増える
- 強化できる
- セーブされる

スマホ確認:

- 縦画面でUIが崩れない
- 仮想スティックが反応する
- 移動しながら掃除ボタンを押し続けられる
- 掃除、回避、状況アクションを押し間違えにくい
- 主要タップ領域は64px以上、その他は48px以上を目安にする
- タッチ操作中にブラウザスクロールが起きない
- iPhoneのホームインジケータなどセーフエリアに主要操作がかぶらない

PC確認:

- `W / A / S / D` と方向キーで移動できる
- `Space` で掃除できる
- `Shift` で回避できる
- `E` で状況アクションを実行できる
- `Esc` でメニューを開ける
- ブラウザサイズ変更に耐える

公開確認:

- `npm install`
- `npm run dev`
- `npm run build`
- Cloudflare Pagesのビルド設定で公開できる

## リスクと対策

- 画像のような高密度アートを最初から完全再現しようとすると重くなるため、MVPでは軽量なドット絵風表現に絞る
- 操作が複雑になるとスマホで遊びにくくなるため、移動、長押し掃除、回避、状況アクション、メニューに絞る
- UIの見た目と当たり判定がずれると誤操作が増えるため、表示オブジェクトとインタラクティブ領域を同じ座標設計にする
- 旧サイトの不要ファイルや文言が混ざると公開品質が落ちるため、ゲームMVPに不要な要素は持ち込まない
- PWAキャッシュが古いファイルを残す可能性があるため、公開前にキャッシュ名と更新手順を確認する
