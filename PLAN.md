# 副業タイプ診断 運用方針

## 公開構成

- Cloudflare Pagesと独自ドメインで公開する
- 診断結果の要約直後に、タイプ別の成果報酬広告を最大3件表示する
- AdSenseは公開時に申請し、承認後は診断結果の末尾に手動広告を1枠表示する
- 運営者情報、問い合わせ、プライバシーポリシー、免責事項はフッターから案内する

## 広告運用

- ASPで提携承認された案件だけを有効化する
- 広告カードには「PR」、広告欄には「広告・PRを含みます」と表示する
- 診断結果は広告案件の有無によって変更しない
- サービス名、特徴、CTA、リンク先が実際の案件内容と一致しているか定期的に確認する
- AdSenseは自動広告を使わず、診断操作やASP広告と混同しない位置に置く

## コンテンツ運用

- 公開後は4タイプそれぞれの始め方、比較、注意点を扱う記事を追加する
- 公式情報や公的情報を確認し、更新日と内容の古さを定期的に見直す
- 収益や成果を保証する表現は使わない

## 公開前チェック

- `config.js` の本番設定
- `robots.txt` と `sitemap.xml` のドメイン置換
- `ads.txt` のpublisher ID設定
- Cloudflare Pagesのシークレット設定
- 全4タイプの診断、広告、問い合わせ、スマートフォン表示の確認
