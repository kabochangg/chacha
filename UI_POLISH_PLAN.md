# 現行UI流用の画面仕上げ計画

## 方針

- 現行の暗いダンジョン調、金枠、四角いタブ、丸い操作ボタンは維持する。
- 新規大型グラフィック制作ではなく、文字の収まり、スマホでの読みやすさ、操作しやすさを優先して磨く。
- v1公開版として、短時間で遊びやすく、文字が読めて、操作に迷わない状態を目指す。

## 修正対象

- 拠点メニューは、ヘッダー、本文、下部メッセージ、タブバーの衝突を避ける。
- ホーム、持ち物、マップ、依頼、ショップは、見出しと本文が重ならないようにする。
- 長文は短く自然な日本語にし、スマホ幅で画面外にはみ出さないように改行する。
- プレイ画面は、現行のマップ、キャラ、残骸、敵、視線扇形、操作ボタンの構成を維持する。
- HUDの文字は短くし、HP、STM、素材、清掃率がスマホで読めるようにする。
- 操作ボタンは「掃除」「回避」「払う」を基本にし、主要ボタンの大きさと同時押ししやすさを優先する。
- PWA公開版で古いキャッシュが残り、文字や画面が古いままになる問題を避ける。

## 確認項目

- タイトル、拠点ホーム、持ち物、マップ、依頼、ショップ、ダンジョン、一時停止、リザルトをスマホ縦画面で確認する。
- PC操作の WASD、Space、Shift、J、Esc が破綻しないことを確認する。
- `npm.cmd run build` が成功することを確認する。
- 公開/PWA版で古いキャッシュにより壊れた表示が残らないことを確認する。
