# v1画面仕上げ・グラフィック制作計画

## 方針

- 現行の暗いダンジョン調、金枠、四角いタブ、丸い操作ボタンは活かす。
- 完成形へ向けて、画面の密度、視認性、押しやすさ、報酬の気持ちよさを順に磨く。
- 新規大型グラフィック制作ではなく、Phaser図形描画、生成テクスチャ、軽量なドット絵風パーツで作る。
- 添付イメージは方向性として扱い、画像そのものは素材として直貼りしない。
- スマホ縦画面で読めて軽く動くことを、見た目の豪華さより優先する。

## 制作単位

### タイトル

- 暗い石畳、ランタン光、床の汚れで「勇者が去った後」の空気を出す。
- ロゴ周りは大きく読みやすくし、ゲーム名と副題が最初の視線に入るようにする。
- `はじめから`、`つづきから` は押しやすい大きさを維持し、金縁と短い押下演出で反応を出す。
- 下部の補足文は短くし、スマホ幅で文字が詰まらないようにする。

### 拠点

- 上部リソースバーで所持金、素材数、出動回数、強化状態を読みやすくする。
- 中央パネルは羊皮紙/木枠寄りの質感を足し、本文とボタンが重ならない余白を守る。
- 下部タブは `ホーム`、`持ち物`、`マップ`、`依頼`、`ショップ` の役割が分かるように、選択中の差分を強める。
- 倉庫は下部タブを増やしすぎず、拠点または `持ち物` から入れる導線にする。
- 依頼やショップは、次にダンジョンへ行く理由が自然に分かる短い文言にする。

### ダンジョン

- 床と壁はタイル差分、縁、影で暗い地下道らしさを出す。
- 残骸は種類ごとに形を変え、掃除対象だと一目で分かるサイズにする。
- 素材は小さくても拾えるものだと分かるよう、軽い浮遊、輪郭、回収時の吸い込みを入れる。
- 危険物は赤系、影、揺れ、発見 `!`、追跡挙動で「避ける/払う」対象だと伝え、視線扇形は表示しない。
- 出口は条件達成後にランダムな到達可能位置へ出現させ、緑の光、矢印、短い揺れで帰還場所として目立たせる。
- プレイヤーは清掃員らしい帽子、バッグ、ほうきのシルエットを保ち、背景に埋もれない輪郭にする。

### HUDと操作

- HUDはHP、STM、素材、清掃率、階層を短い文言で表示する。
- ミニマップ風パネルは、階層感と現在地の目安を伝える範囲に留める。
- 清掃リング、粉じん、短い揺れ、掃き音で長押し掃除の手応えを強める。
- 仮想スティック、掃除、走る、払う、メニューは、左右反転後も同時押ししやすい位置にする。
- 主要タップ領域は64px以上、その他は48px以上を目安にする。

### リザルト

- 清掃率、素材、売上、倉庫収納、所持金、強化までの距離、作業時間を上から順に読みやすく並べる。
- 回収素材は換金するか倉庫へ収納するかを選べるようにし、拠点でも倉庫内素材を売る/保管する整理導線を置く。
- 素材や売上が増えたことを、短い光、粒、音で気持ちよく見せる。
- `もう一回行く` を最優先ボタンとして大きく配置し、次の出撃へ自然につなげる。
- `拠点へ戻る` は強化や生産に向かう導線として、報酬情報の近くで意味が分かるようにする。

## 優先順位

1. スマホ縦画面で文字、ボタン、HUDが重ならないようにする。
2. タイトル、拠点、ダンジョン、リザルトの第一印象を同じ世界観に揃える。
3. 掃除中、素材回収、危険物を払う、出口開放、報酬獲得の演出を磨く。
4. 図鑑、持ち物、倉庫、依頼、ショップを空白に見せず、最低限の情報を見やすくする。
5. 実機タッチ確認後に、押しづらい位置、セーフエリア、長押し誤解除を直す。

## 確認項目

- タイトル、拠点ホーム、持ち物、倉庫、マップ、依頼、ショップ、ダンジョン、一時停止、リザルトをスマホ縦画面で確認する。
- 左右どちらの操作レイアウトでも、移動しながら掃除、走る、払う操作ができることを確認する。
- 危険物の視線扇形が表示されず、発見 `!`、追跡状態が背景に埋もれないことを確認する。
- 素材回収と報酬獲得が短い演出で分かることを確認する。
- `npm.cmd run build` が成功することを確認する。
- 公開/PWA版で古いキャッシュにより壊れた表示が残らないことを確認する。
